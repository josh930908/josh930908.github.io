return b + ((a % 2)? recur(a - 1, a/2 + b) : ((a % 3)? recur(a - 1, a/3 + b) : ((a % 5)? recur(a - 1, a/5 + b) : recur(a - 1, a + b)))); } ``` 試問 $recur(8,7)$ 的回傳值為何？ ``` (A) 111 (B) 108 (C) 106* (D) 我不想算了~ ``` ## 第十六題 精通**遞迴**的你們，一定知道有密切關聯的**分治法**。 現在 Ducky 無聊想用分治法求區間和： ```cpp int arr[] = {123, 576, 874, 238, 516, 867, 52, 913, 870, 153, 841, 534, 365, 480}; } ``` 請問 $easyBitwiseOperation(5)$ 會輸出？ ``` (A) 5 (B) 4* (C) 0 (D) 我懶得看ㅍ_ㅍ ``` ## 第十九題 卓越的*黃瓜* 提醒天真的 Ducky，選手們 ~~(尤其是*蘿糕* 的)~~ 可能會偷偷把程式碼抄到 IDE 裡面執行。 於是 Ducky 要再出一題來檢驗選手是否真的看懂了。 ```cpp void mysteriousBitwiseOperation(int n, int m){ cout << ((((n & 1023) << (m % 10)) | ((n & 1023) >> (10 - m % 10))) & 1023); } ``` 若**忽略C++整數存取範圍限制**，$mysteriousBitwiseOperation(2^{100}-2^{10}-3,\ 111)$ 之輸出為？ $(A)\ 2^{100} - 2^{10}-3$ $(B)\ 2^{100} - 2^{10}-4$ $(C)\ 2^{10} - 4$ $(D)\ 2^{10} - 5$* *簡答: 10-bit left bit rotation* ## 第二十題 寫完前面十九題，有沒有覺得人生一片灰暗？沒關係的，第二十題是一個相當簡單的題目，考驗邏輯思考能力。 有一個相當經典的博奕論：海盜分金問題。題目是這樣的： > 有A,B,C三個海盜，他們得到了100個金幣，卻為了如何分錢感到困擾。為彰顯民主，他們決定辦一場三個人的小投票，依次由A,B,C提出一種分錢的方法，如果同意票數**超過**當前人數的一半，就依照該方法實施，否則就將提出方案的人丟去餵鯊魚。而海盜們會在**保證自身性命的前提下做出對自身最有利的決定**。則請問下列敘述何者正確？ ``` (A)海盜A能夠活命，不過拿不到金幣 (B)最大受益者為海盜B (C)為保證自身安全，海盜A提出的方案會使海盜B受益 (D)海盜B拿不到任何金幣 ```
